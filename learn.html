Linux C/C++ 后台服务器开发学习

1.C/C++学习

1.1 《C++ Primer》

1.2 《STL 源码剖析》和《深度探索 C++ 对象模型》

指针、引用、数组、内存

引用与指针区别

C 和 C++ 的一些区别，比如 new、delete 和 malloc、free 的区别

虚机制：虚函数、虚函数表、纯虚函数

多态： 动态绑定，静态多态

重写、重载

智能指针使用：shared_ptr、weak_ptr、unique_ptr等

一些关键字的作用：static、const、extern

STL部分容器的实现原理，如 vector、deque、map、hashmap

编译链接机制

除此之外还有一些新特性，比如右值引用、简单的智能指针等，源码剖析建议看候捷老师的视频，第一本书我看的是第五版，内容还是比较多的。

2.计算机网络学习

2.1 掌握网络协议和知识

HTTP、TCP、IP、ICMP、UDP、DNS、ARP

IP地址、MAC地址、OSI七层模型（或者 TCP/IP 五层模型）

HTTPS安全相关的：数字签名、数字证书、TLS

比如，为了区分同一个主机的不用应用，引入了传输层，并使用不用的端口号作为区别； 为了在不同子网间传输数据引入了网络层，并使用 IP 地址寻址路由;网络层解决了不同子网间路由的问题，但是同一个局域网内确定主机却是通过 MAC 地址，所以引入了链路层来承载 IP 数据包; 同时为了将 IP 地址和 MAC 地址做转换映射又产生了 ARP 协议。

2.2 抓包真正理解握手等过程

三次握手、四次挥手

状态转换

TCP 状态中 TIME_WAIT

拥塞控制

TCP 连接建立为什么不是两次握手

TIME_WAIT 的存在解决了什么问题，等待时间为什么是 2 MSL

路由器和交换机有什么区别？

这里推荐一本书《图解HTTP》

linux环境下可以直接用tcpdump抓包，windows可以下载Wireshark，对于tcp相关操作我主要使用tcpdump,理解https的ssl层安全握手， 我主要用的Wireshark，正好课表安排中学过网络安全课程，理解起来更是得心应手，建议可以先了解网安的知识再看https。

3.unix&linux内核（操作系统）

3.1 内存管理

虚拟内存，段页式，内部外部内存碎片等问题

3.2 进程与线程

老生常谈的问题了，看个人的理解，其实二者底层都是do_fork()调用的，只是共享参数不同

3.2.1 进程间通信IPC

3.2.2 多线程同步/互斥

3.2.3 死锁

3.2.4 悲观锁乐观锁

3.3 调度算法

3.3.1 进程调度算法，比如先来先服务，时间轮片法，最短作业优先等等，简单了解

3.3.2 页面调度算法，比如先进先出，最近最久未使用等等，简单了解

4.网络编程和系统编程

主要是系统编程和网络IO，我简单总结一下，具体还需要实践，学完了可以研读muduo源码，他主要是做的静态链接库，基于对象编程，很有启发！！

进程间通信方式： 信号量、管道、共享内存、socket 等

多线程编程：互斥锁、条件变量、读写锁、线程池等

IO 模型：同步、异步、阻塞、非阻塞...

高性能 IO 两种模式：Reactor 和 Proactor（ 但是 Linux 下由于缺少异步 IO 支持，基本没有 Proactor

IO 复用机制：epoll、select、poll（破解 C10K 问题的利器）

书籍推荐

《Linux多线程服务器端编程》陈硕写的非常难懂，《Linux高性能服务器编程》这本书入门不错

5.数据库

数据库我用得少，了解的不深，简单列举

索引存储结构：B树、B+树索引、Hash索引

索引的使用：主键索引、覆盖索引、最左前缀原则、索引下推等

锁：乐观锁、悲观锁、表锁，行锁，意向锁，读锁，写锁等等

事务：ACID理论

MySQL InnoDB MVCC 实现机制，这个机制其实就是保证可以同时写，但是具体过程还是比较冗杂的

6.数据结构与算法

数组、链表

栈与队列

树、二叉树、多叉树实现和遍历方式，红黑树（五个性质，旋转变色）

B树、B+树，为什么可以用来在数据库

深度优先遍历dfs和广度优先遍bfs历（涉及到递归、回溯）

常见的算法

排序算法：冒泡、插入、快速、堆排等，我主要记了这几个经典的思路

字符串匹配算法：KMP，一般存一个next数组，保存最长公共前后缀。前缀是指不包含最后一个字符的所有以第一个字符开头的连续子串。 后缀是指不包含第一个字符的所有以最后一个字符结尾的连续子串。

常见算法思想：递归、枚举、递推、分治、贪心、动态规划等，LeetCode很多中等苦难全是用的这几个
